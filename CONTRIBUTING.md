# Contributing to erp-marketplace-ai

Thank you for considering contributing to the `erp-marketplace-ai` project! We welcome contributions from the community. Here's how you can help.

## Guidelines for Contributing

1. **Bug Reports**  
   If you find a bug in the project, please open an issue with a clear description and steps to reproduce it. 

2. **Feature Requests**  
   If you have an idea for a new feature, feel free to open a discussion or create an issue to propose your feature.

3. **Code Contributions**  
   - Fork the repository and create your branch from `main`.
   - Make sure your code follows the style guide and is well-documented.
   - Write tests to cover your changes.
   - Ensure all tests are passing.

## Code Standards

- Follow the JavaScript style guide (e.g., Airbnb or StandardJS).
- Use `async`/`await` for asynchronous code instead of callbacks.
- Document your code using JSDoc format.
- Adhere to DRY (Don't Repeat Yourself) principles and SOLID principles of object-oriented design.

## Testing

- Write tests for your code changes.
- Ensure that your tests are robust and cover edge cases.

## Pull Request Process

1. **Check the Pull Request Checklist**  
   - [ ] Code follows the style guide
   - [ ] Code is tested and all tests pass
   - [ ] Documentation is updated (if applicable)

2. **Code Review Process**  
   - After submitting a pull request, it will be reviewed by maintainers.
   - Be open to feedback and make necessary adjustments.

Thank you for contributing! Happy coding!
